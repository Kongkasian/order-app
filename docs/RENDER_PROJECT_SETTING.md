# Render Project 설정 가이드

## Project (Optional) 설정

Render에서 Web Service를 생성할 때 "Project (Optional)" 옵션이 있습니다.

### 선택지

1. **No / None** (권장 - 단일 서비스인 경우)
   - 프로젝트 그룹에 속하지 않음
   - 독립적인 서비스로 관리
   - 처음 배포할 때는 이 옵션을 선택하세요

2. **New Project** (여러 서비스를 그룹화할 때)
   - 새 프로젝트를 생성하여 서비스들을 그룹화
   - 예: "order-app" 프로젝트에 백엔드와 프런트엔드를 함께 관리
   - 나중에 프런트엔드도 배포할 때 같은 프로젝트에 추가 가능

### 권장 사항

**처음 배포할 때:**
- **Project (Optional)**: `No` 또는 `None` 선택
- 나중에 프런트엔드도 배포할 때 같은 프로젝트로 묶을 수 있음

**프로젝트 그룹화가 필요한 경우:**
1. 먼저 "New Project"로 프로젝트 생성 (예: "order-app")
2. 백엔드 서비스를 해당 프로젝트에 배포
3. 나중에 프런트엔드도 같은 프로젝트에 추가

### 결론

**처음에는 `No` 또는 `None`으로 설정하세요!**

이것은 선택사항이므로 서비스 동작에는 영향을 주지 않습니다. 단지 대시보드에서 서비스들을 그룹화하여 관리하기 위한 기능입니다.
